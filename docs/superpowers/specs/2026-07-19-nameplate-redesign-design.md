# 대하산업기계 홈페이지 리디자인 — 「명판」 설계서

작성일: 2026-07-19
브랜치: `redesign-nameplate`

## 1. 목표와 범위

### 목표
낯선 **신규 발주처의 구매·기술 담당자**를 **견적요청**까지 데려간다. 이 문장이 모든 설계 판단의 기준이다. 어떤 요소가 이 경로에 기여하지 않으면 뺀다.

### 범위에 포함
- 전체 비주얼 아이덴티티 재설계 (팔레트·타이포·레이아웃 전면 교체)
- 26개 페이지 전부의 화면 재구성
- 메뉴 구성과 페이지 목록은 **현행 유지** (추가·삭제·이동 없음)
- CSS를 `styles.css`로 분리

### 범위에서 제외 (합의됨)
- **스펙 데이터의 HTML 전환.** 토출량·압력·kW 표와 성능곡선은 지금처럼 이미지로 둔다.
- **신규 사진 촬영.** 기존 `images/gallery/` 누끼 컷으로만 설계한다.

### 제외에 따른 한계 (명시)
이 두 가지가 빠진 상태에서 이번 작업이 도달할 수 있는 거리에는 한계가 있다. 화면의 격과 신뢰도는 확실히 올라가지만, 견적요청 전환을 제대로 밀어올리려면 결국 (1) 제원 데이터의 웹 네이티브화와 (2) 자사 제품·공장 사진이 필요하다. **이번 구조는 그 둘이 나중에 들어올 자리를 비워둔 채로 설계한다** — 제품 상세 템플릿(T2)의 이미지 슬롯은 나중에 HTML 표로 교체 가능한 단위로 끊어둔다.

## 2. 현황 진단

### 구조
- `index.html` 단일 파일 104KB. `goto()`로 `<div class="page">`를 전환하는 SPA 방식.
- 26개 페이지가 한 파일 안에 있음. CSS 약 750줄이 `<style>`에 인라인.
- 기존 팔레트: 네이비 `#0a1628` + 골드 `#c8922a`. 서체: Noto Sans KR + Barlow Condensed.

### 자산 실태 (조사 결과)
| 문제 | 대상 | 조치 |
|---|---|---|
| 자사 제품이 아닌 스톡 사진 | `product-photo-01.jpg` (초고진공 챔버) | **참조 제거** (파일 자체는 보존) |
| 파일명과 내용 불일치 | `plungerpump-photo.jpg`(실제 스펙표), `washer-photo.jpg`(실제 노즐선정표) | 용도에 맞게 재배치 |
| 카탈로그 PDF 페이지를 통째로 래스터화 | `ringblower-0*.jpg`, `washingsystem-specs*.png` (개당 2~3MB) | 명판 프레임 + 라이트박스 확대 + lazy load |
| 유일하게 쓸 만한 사진 자산 | `images/gallery/` 흰 배경 누끼 컷 | **설계의 주인공으로 사용** |
| 인쇄물 CI 불일치 | 링블로워 카탈로그=파랑, 세척기 카탈로그=초록, 웹=네이비 | 지켜야 할 단일 CI 없음 → 새 아이덴티티에 유리 |

### 작업 시작 시점의 미커밋 변경
상단 유틸 영역의 `INTRANET` 링크 1줄이 제거된 상태였다. 의도적 결정으로 판단하여 **새 디자인에도 포함하지 않는다.**

## 3. 디자인 방향 — 「명판」

### 근거
산업기계에는 반드시 명판이 붙는다. 모델명·압력·유량·출력·제조사·제조일이 금속판에 각인되어 리벳으로 고정된 것. 이 업계에서 "이 숫자를 우리가 책임진다"는 뜻의 물건이다.

이 회사의 핵심 주장이 **"100% 자체 설계·국내 생산"**이고, 쓸 수 있는 사진이 흰 배경 표본 컷뿐이라는 두 조건이 같은 답을 가리켰다: **회사가 자기 제품과 똑같은 방식으로 자신을 소개한다.** 큰 사진 없이도 밀도와 신뢰가 서고, 26개 페이지에 일관되게 적용되며, 무엇보다 이 회사에 정직하다.

### 검토했으나 채택하지 않은 방향
- **유동(Flow field)** — CFD 유동장을 디자인 언어로. 그라디언트+모션 주도는 AI 기본 산출물이 몰리는 지점이고, 쓸 수 있는 제품 컷이 3장뿐이라 추상 그래픽이 화면을 채우게 된다. 개인정보처리방침 페이지에까지 유동장을 깔 수 없다는 확장성 문제도 있다.
- **제원(도면 시트)** — 치수선·지시선 중심의 임상적 방향. 정확하지만 견고함을 파는 회사에 비해 차갑고, 헤어라인 괘선 조판형 레이아웃은 알려진 AI 기본값에 인접하다. **제품 상세(T2)에 한정해 부분 차용한다.**

### 자기 비평 (기록)
팔레트("산업용 회색 + 녹슨 빨강")는 이 안에서 가장 예측 가능한 부분이며 여기서 독창성이 나오지 않는다. 이 디자인을 대하산업기계 것으로 만드는 것은 **명판이라는 문법 자체**와 회사를 명판으로 제시한다는 아이디어다. 색은 그것을 방해하지 않는 역할까지만 한다.

## 4. 토큰 시스템

### 4.1 색 — 기계 표면에서 추출

```css
--cast:     #23262A;  /* 주철 — 잉크, 다크 그라운드 */
--iron:     #565D63;  /* 기계 회색 — 보조 텍스트, 괘선 */
--machined: #C9CDD0;  /* 가공면 — 경계, 플레이트 테두리 */
--plate:    #ECEDEB;  /* 명판 바탕 — 페이지 그라운드 */
--oxide:    #B23A1E;  /* 산화 적색 — 단일 액센트 */
```

**액센트 사용 규칙 (엄수):** `--oxide`는 다음 세 곳에만 나타난다.
1. 견적요청 CTA
2. 현재 위치 표시 (활성 내비 항목, 활성 탭)
3. 카테고리 마크 (페이지 명판 상단의 작은 표식)

그 외 모든 요소는 회색조. 화면에서 가장 강한 색이 이 사이트가 존재하는 이유(견적요청)와 일치해야 한다.

**명판을 밝게 가는 이유:** 어두운 배경 + 단일 강한 액센트는 AI 산출물이 몰리는 패턴이다. 동시에 실제 산업 명판은 스테인리스·알루미늄 판에 어둡게 각인하는 것이 정석이다. 회피와 사실이 같은 답을 가리켰다.

### 4.2 타이포 — 3개 역할

| 역할 | 서체 | 용도 |
|---|---|---|
| Display | **Archivo** 700–800, 대문자, 타이트 트래킹 | 각인된 표제, 영문 제품명 |
| Body / KR | **IBM Plex Sans KR** 400/500 | 국문 본문 전체 |
| Data | **IBM Plex Mono** 500, tabular figures | 모델코드, 제원 수치, 명판 데이터행 |

**선택 근거:** 국문에 Pretendard는 국내 사이트 기본값에 가까워 제외. IBM Plex는 엔지니어링 문서 계열이고, 무엇보다 **Plex Sans KR과 Plex Mono가 한 가족**이라 국문 본문과 제원 숫자가 같은 혈통으로 붙는다. 기존 Barlow Condensed는 재사용하지 않는다.

**규칙:** 모델코드와 제원 수치는 예외 없이 고정폭 + tabular figures로 조판한다. 표를 눈으로 훑는 기술 담당자에게 자릿수 정렬은 장식이 아니라 기능이다.

### 4.3 레이아웃 — 플레이트 시스템

기본 단위는 **명판(plate)**: 경계가 분명한 사각형 + 상단 밴드(카테고리/영문 라벨) + 각인선 + 데이터 영역.

- 12컬럼 그리드. 단, 플레이트는 성긴 모듈에 스냅시켜 페이지가 흐르는 카드가 아니라 **조립된 패널**로 읽히게 한다.
- 리벳(모서리 4점)은 **홈 히어로 명판에만** 사용. 나머지 플레이트에는 붙이지 않는다. 스큐어모픽 키치로 기우는 위험이 가장 큰 요소이므로 최소 사용.
- 질감·그림자·그라디언트 없음. 구조와 경계선으로만 명판감을 만든다.

### 4.4 시그니처 — 회사 명판 히어로

홈 히어로는 **대하산업기계 자신의 명판**이다. 스톡 사진을 대체하며, 사진 없이 신뢰를 세우는 핵심 장치.

```
┌─────────────────────────────────────────────┐
│ ●  NAMEPLATE              DHFM · INCHEON KR ●│
├─────────────────────────────────────────────┤
│                                             │
│   대하산업기계                                │
│   주식회사                                    │
│   DAEHA INDUSTRIAL MACHINERY CO., LTD.      │
│   ═══════════════════════════════════════   │
│   MFG.     링블로워 · 고압플런저펌프 · 세척시스템 │
│   EST.     2016                             │
│   LOC.     인천광역시 서구 봉수대로 1568번길 63-20│
│   CERT.    ISO 9001                         │
│   ORIGIN   100% 자체 설계·국내 생산            │

│ ● EST. 2016 — INCHEON      [ 견적요청 → ]  ● │
└─────────────────────────────────────────────┘
```

### 4.5 모션

**하나의 순간만 둔다.** 페이지 로드 시 히어로 명판의 데이터행(MFG./EST./LOC./CERT./ORIGIN)이 위에서부터 순차로 채워진다. 총 600ms.

그 외에 움직이는 것은 호버 상태뿐이다. `prefers-reduced-motion: reduce`에서는 전부 즉시 표시한다. 산업재 사이트에서 산만한 애니메이션은 신뢰를 깎는다.

## 5. 홈 구성

| # | 섹션 | 내용 |
|---|---|---|
| 01 | 회사 명판 (히어로) | 4.4 참조. CTA: `견적요청`(주) / `제품 보기`(부) |
| 02 | 주력 제품 3종 | 플레이트 그리드. 갤러리 누끼 컷 + 대표 수치 3줄(고정폭) |
| 03 | 근거 4항 | 얇은 명판 띠. 2016 / ISO 9001 / 100% 국산 / 5축가공·유동해석 (아래 주 참조) |
| 04 | 사업분야 3종 | 가로 띠, 링크만 |
| 05 | 견적요청 밴드 | 화면에서 유일하게 강한 색(`--oxide`)을 쓰는 지점 |
| 06 | 연락처 + 네이버지도 | 주소·전화·팩스·이메일. **폼 없음** |

### 근거 4항의 내용 변경 (확정)
현재 홈의 stat box는 `2016 / ISO / 100% / A/S`다. 이 중 마지막 항목을 **`A/S 신속대응` → `5축가공·유동해석`으로 교체한다.** 근거: 앞의 세 항목이 모두 검증 가능한 사실인 반면 "A/S 신속대응"은 모든 업체가 하는 주장이라 변별력이 없다. 반면 5축 가공기 보유와 유동해석 기반 설계는 카탈로그에 근거가 있고 경쟁사 대비 실제 차별점이다. A/S는 제품 상세(T2)와 사업분야(T3) 본문에서 구체적으로 다룬다.

### 홈에서 문의 폼을 제거하는 이유
현재 폼이 홈 하단·견적요청·온라인상담·Contact 네 곳에 중복 존재한다. 방문자는 어디에 써야 할지 모르고, 관리자는 문의가 네 갈래로 흩어진다. **홈은 견적요청으로 보내는 역할만** 하고, 폼은 견적요청 페이지에 한 벌만 제대로 둔다. (온라인상담·Contact 폼은 페이지가 유지되므로 존치하되, 홈에서의 진입 동선은 견적요청으로 단일화한다.)

### 제품 카드의 대표 수치
스펙 표 HTML화는 범위 밖이지만, 제품별 **범위값 한 줄**(최대 압력 / 최대 유량 / 출력 범위)은 카탈로그 이미지에서 판독하여 카드에 직접 텍스트로 쓴다. 홈에서 "내가 찾는 급인지" 판단이 끝나게 하는 것이 목적이다.

- 구현 시 `images/` 내 카탈로그 이미지를 판독하여 4개 제품의 범위값을 추출한다.
- 판독으로 확인된 예: 플런저펌프 F5000/F6000 계열 — 최대 200 bar, 최대 1717 L/min, 최대 530 kW.
- **추출한 모든 수치는 게시 전 대표님 검수를 받는다.** 검수 전에는 실서비스에 반영하지 않는다.

## 6. 페이지 템플릿 8종

메뉴 구성과 페이지 목록은 현행을 그대로 유지하며, 26개 페이지를 8개 템플릿으로 찍어낸다.

| 템플릿 | 적용 페이지 | 수 | 구성 |
|---|---|---|---|
| T1 홈 | 홈 | 1 | 5장 참조 |
| T2 제품 상세 | 링블로워 · 플런저펌프 · 세척기 · 세척시스템 | 4 | 누끼 컷 + 특징 플레이트 + 카탈로그 이미지(라이트박스) + 견적요청 CTA. 도면 시트 어법(치수선·지시선) 부분 차용 |
| T3 사업분야 | 블로워 · 펌프 · 펌프시스템 | 3 | 이미지 스택 중심, 명판 프레임 |
| T4 문서 | CEO인사말 · 경영방침 · 개인정보 · 이용약관 · 채용안내 · 인재상 · 복리후생 | 7 | 단일 컬럼 본문. 읽기 폭 제한, 각인선으로 절 구분 |

| T6 폼 | 견적요청 · 온라인상담 · Contact | 3 | 명판형 폼. 라벨 상시 노출, 필수 표시 명확 |
| T7 갤러리 | 제품사진 | 1 | 탭 + 그리드 + 라이트박스 (기존 구조 유지, 외형 교체) |

**모든 페이지 공통:** 상단에 **페이지 명판**으로 시작한다 — 카테고리 마크(`--oxide`) + 페이지명 + 각인선. 현재의 `.page-banner`를 대체하며, 26개 페이지가 같은 리듬으로 열린다.

## 7. 카탈로그 이미지 처리

스펙 HTML화는 범위 밖이지만, 디자인 범위 안에서 가능한 개선을 적용한다.

1. **명판 프레임에 넣는다** — 이미지가 페이지에 그냥 박히지 않고 경계와 라벨을 갖는다.
2. **클릭 시 라이트박스 확대** — 갤러리용 라이트박스를 재사용한다. 모바일에서 최소한 읽을 수는 있게 된다.
3. **`loading="lazy"` 적용** — 개당 2~3MB 이미지가 초기 로딩을 막지 않게 한다.
4. **명시적 `width`/`height` 지정** — 레이아웃 시프트 방지.

데이터는 여전히 이미지지만, 접근성과 초기 로딩은 실질적으로 개선된다.

CSS를 100% 재작성하므로 분리 적기다. JS는 규모가 작아 `index.html`에 인라인 유지한다. 빌드 단계 없음 — FTP 업로드 그대로 가능.

### 유지하는 것
- `goto()` 기반 SPA 페이지 전환 방식
- 네이버 지도 연동
- 폼 전송 로직
- 갤러리 라이트박스
- SEO 메타 태그, Open Graph, JSON-LD 구조화 데이터 (내용 검토 후 유지)
- 모바일 햄버거 메뉴 + 아코디언 서브메뉴

### CSS 작성 규칙
기존 파일에서 타입 기반 선택자(`.section`)와 요소 기반 선택자(`.cta`)가 서로 상쇄되는 문제가 관찰된다. 특히 섹션 간 padding/margin에서 발생하기 쉽다.

- 선택자 특정성을 단일 계층으로 유지한다. 중첩 깊이 2를 넘기지 않는다.
- 섹션 간격은 단일 유틸리티로 통일하고 개별 재정의를 금지한다.
- 토큰은 `:root`에 정의하고 하드코딩된 색·크기 값을 두지 않는다.

## 9. 품질 기준 (구현 완료 조건)

- **반응형** — 모바일(375px)까지 가로 스크롤 없이 동작
- **키보드 접근성** — 모든 대화형 요소에 보이는 포커스 링. 내비 드롭다운 키보드 조작 가능
- **모션 존중** — `prefers-reduced-motion: reduce`에서 애니메이션 제거
- **이미지** — 전부 `alt` 보유, `loading="lazy"`, 명시적 치수
- **대비** — 본문 텍스트 4.5:1 이상, `--oxide` CTA의 흰 글자 대비 확인
- **검증** — 26개 페이지 전부 실제 브라우저에서 확인. 데스크톱·모바일 폭 양쪽 스크린샷 확보
